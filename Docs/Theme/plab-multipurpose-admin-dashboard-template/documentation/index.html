Plab - Multi-Framework Admin Documentation

Introduction

Plab is a Responsive, Clean and Minimal Multi-Framework Admin Dashboard template. The template contains admin dashboard versions for Laravel, Yii, Ruby on Rails, Angular 9, React, React Next and Bootstrap 4. Plab comes with a mega collection of features which can make admin dashboard of any projects like Small to Medium and Large sizes of web or mobile apps including Personal projects, eCommerce, CRM, ERP, SaaS, PaaS, CMS, Back office and Accounting Software backend etc.

The Multipurpose feature of the dashboard template will allow integrating for different types of business projects. Plab offers a creative and easy way to develop the admin dashboard backend for your next exciting projects.

React Version

In order to kickstart the development with Plab, the very first thing need to do is to setup a react development environment.

Following tools are needed to setup a react dev environment:

  • Node JS

    • Node JS works as a node package manager behind the scene. It is recommended to download and install the latest version from the Node JS official site http://nodejs.org/.

  • Yarn

    • Download the latest version of the most loved dependency management tool for react applications from its official site https://yarnpkg.com/ and install it on your machine.

Preparation

You'll need to install Node.js >=v8.11.4 (Recommended Version) (NPM comes along with it) and serve (a NPM package)

Use command line tool i.e. powershell or terminal and navigate to the project root. This is powershell in windows and terminal on mac/linux.

Then run : npm install and this will import all the dependencies for the Plab React Admin.

After all the dependency libraries imported successfully then run : npm start and this will return something like the following:

Serving!

- Local: http://localhost:3000

- On Your Network: http://192.168.1.18:3000

If the npm start command works successfully then start checking the Plab by copying the URL http://localhost:3000 to the browser.

Build the Project

Once all updating done then you might have to make a build to reploy the application to the online server. To do that, you could try following steps:

Run to build the project : npm run build. The build artifacts will be stored in the build/ directory.

Static Server

For environments using Node, the easiest way to handle this would be to install serve and let it handle the rest:

npm install -g serve

serve -s build

The last command shown above will serve your static site on the port 5000.

It'll return something like this:

Serving!

- Local: http://localhost:5000

- On Your Network: http://192.168.1.18:5000

    const currentSideMenu = this.state.sideMenu;

Changing Layout

The template left sidebar menu default is dark layout. If you want to change this to light default then have to follow below steps:

Step 1:

In the Navigation: (which is placed in src/components/Navigation/Navigation.js)

Comment out the import statement line-

import SideMenuDark from './SideMenu/SideMenuDark';

Step 2:

Remove the menuColor state from state.

Step 3:

Remove the arrow class which is in onSideMenuHandler.

Step 4:

Replace the SideMenuLight component with ternary JSX syntax from below of Navigation class.

Step 5 & finally:

Remove the Switcher component ColorSwitch from below of Navigation class.

Laravel 7 Version

Plab Laravel version can be used directly from the plab-laravel project folder or by copying few parts from the downloaded template folder. To install Laravel and creating a new project, we recommend visiting the Laravel official site.

To up and running with the downloaded Plab Laravel version, following steps needs to be done:

Installation
Server Requirements

It is good to have Laravel Homestead virtual machine setup and running for your Laravel project. If you are not using Homestead, you will need to make sure your server meets the following requirements:

  • PHP >= 7.2.0
  • BCMath PHP Extension
  • Ctype PHP Extension
  • JSON PHP Extension
  • Mbstring PHP Extension
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension
Installing Laravel
    Steps:
  • Go to the project root directory i.e. plab-laravel then run
    composer install
  • Before executing the following commands need to rename .env.example to .env anf then continue following steps.. php artisan key:generate
  • npm install to install dependencies.
  • php artisan serve to serve the application.
  • npm run watch to watch all the changes you make in the application
  • npm run dev to recompile after making any changes in dev mode.
  • npm run prod to recompile for the production deployment.
    • If needed any permission then
  • sudo chmod -R o+rw bootstrap/cache
  • sudo chmod -R o+rw storage
    • Check Laravel official site. for more details.

  • php yii serve to install dependencies.
    • Check Yii official site. for more details.

Installation

If you do not already have Ruby installed, you may do so by following the instructions at rubyinstaller.org.

If you do not already have Rails installed, you may do so by following the instructions (After installing Ruby):

  • bundle install to install dependencies.
    • Then run:
  • yarn install --check-files to install dependencies.
    • Then run:
  • rails db:create to create the default DB.
